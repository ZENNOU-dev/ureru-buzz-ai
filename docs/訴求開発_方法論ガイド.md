# 訴求開発 方法論ガイド

## 本ドキュメントの目的

広告制作フローにおける**訴求開発**の考え方・思考プロセス・最終アウトプットを定義する。
AI（Claude等）がリサーチデータをもとに精度の高いコンセプトとWho/What/Whyを生成する際のガイドラインであり、同時にチームメンバーが方法論を共有するためのドキュメントでもある。

> **次ステップとの接続**: 本フェーズで固めたコンセプト・Who/What/Whyは、後続の「広告企画・開発」フェーズにおいて、コンセプトに一貫した表現（興味の型・構成の型・台本）に落とし込まれる。

---

## 1. 訴求開発の役割

広告制作フローの中で、リサーチ（Phase 2）の後に位置し、**どういう方針で広告を作るか**を整理する工程。

```
リサーチ → [訴求開発] → 広告企画 → 台本制作 → ...
```

訴求開発のアウトプットが固まることで、以降の企画・台本・クリエイティブ制作がブレなくなる。

---

## 2. 最終アウトプット

訴求開発で固めるべき項目は以下の4つ。これらが揃っていればアウトプットとして完成。

| 項目 | 定義 |
|---|---|
| **コンセプト** | 便益×カテゴリーの掛け算で生まれる、世の中にない新しい便益を表すイメージの広がる言葉 |
| **WHO** | どんなターゲットに訴求するか（状態・認知状況・制約条件） |
| **WHAT** | そのWHOに対して何を訴求すれば刺さるか（提供価値・便益・カテゴリー定義） |
| **WHY** | そのWHOにそのWHATがなぜ刺さるのか（強いペイン/願望・欲求ファネル・障壁・想定競合） |

加えて、WHYから導かれる**想定競合**と**差別化軸**を明確化する。

---

## 3. コンセプトの定義

### 3.1 コンセプトの構造

```
コンセプト = 新しい便益 × 商品カテゴリー
```

**「どんな便益を提供する、どんなカテゴリーの商品なのか」** の掛け算で考える。
その掛け算の結果を、世の中にない新しい便益として、**一言でイメージが広がる言葉**にする。

### 3.2 良いコンセプトの条件

- 便益×カテゴリーの掛け算が明確である
- 世の中にまだない新しい便益として認識できる
- 一言で聞いただけで「欲しい」「知りたい」と思える
- イメージが広がる言葉として成立している

### 3.3 具体例

| コンセプト | 便益 | カテゴリー | 新しい便益としての解釈 |
|---|---|---|---|
| **タケコプター** | 竹とんぼサイズの | ヘリコプター | 今までにない小さいサイズのヘリコプターでどこにでも飛べる |
| **どこでもドア** | どこでも行ける | ドア | ドアを開けるだけでどこにでも行ける |
| **疲労回復パジャマ**（TENTIAL） | 疲労回復 | パジャマ | これまでパジャマになかった疲労回復の役割を定義 |
| **飲む化粧水**（水の案件例） | 内側から保湿する | 飲料水 | 水を飲むことで内側から保湿される新カテゴリー |

### 3.4 カテゴリーの再定義

コンセプト策定において、商品をどの商品カテゴリーとして定義するかは極めて重要。
既存のカテゴリーに属さない新しいカテゴリーとして再定義することで、新しい便益が生まれる。

- 水 → 飲む化粧水
- パジャマ → 疲労回復ウェア
- 債務整理 → 利息の乗り換え（債務整理の方法論メモより）

> **注意**: カテゴリーの再定義はWHATの整理とコンセプト策定を**行き来しながら**固まっていく。最初から確定するものではない。

## 4. WHO（ターゲット定義）

### 4.1 WHOの整理軸

WHOは以下の3軸で整理する。

| 軸 | 定義 | 例（疲労回復パジャマ） |
|---|---|---|
| **状態** | どんな生活状態・身体状態にある人か | 朝起きた時に身体のこりや疲れを感じる |
| **認知状況** | その課題に対してどう認識しているか | 疲れを感じているが、パジャマで解決できるとは思っていない |
| **制約条件** | 行動を阻む具体的な制約は何か | マッサージに行く時間がない、高額な健康グッズは買えない |

### 4.2 一つのコンセプトに対して複数のWHOが存在する

コンセプトはひとつでも、そこから派生するターゲットは複数パターンある。

**例: 疲労回復パジャマ**

| WHOパターン | 状態 | 認知 | 制約 |
|---|---|---|---|
| 20〜30代ガジェット好き | 新しい機能性プロダクトに興味がある | 機能性パジャマという新カテゴリーを知りたい | 価格が高すぎると手が出ない |
| 40代以降の疲れやすい主婦 | 日常的に肩こり・疲労感がある | 疲れは仕方ないと諦めている | マッサージや整体に通う時間がない |
| プレゼント購入者 | 大切な人への贈り物を探している | 肩こりグッズやマッサージ機と並列で検討 | ありきたりなものは避けたい |
| 朝の寝違えが多い人 | 寝返りのたびに首や肩を痛める | 枕の問題だと思っている | 高級枕を試したが改善しなかった |

## 5. WHAT（訴求内容の定義）

### 5.1 WHATの整理プロセス

WHATは以下の順序で整理する（ただし行き来しながら精度を高める）。

```
商品特徴の理解（機能・デザイン・オファー・権威性）
    ↓
ユーザー視点での価値変換（5つの価値軸）
    ↓
提供便益シーンの想定
    ↓
商品カテゴリーの再定義
```

### 5.2 5つの価値軸

商品特徴をユーザー視点の価値に変換する際、以下の5軸で整理する。

| 価値軸 | 定義 | 吉野家の例 |
|---|---|---|
| **機能価値** | 商品の機能・性能がもたらす具体的な価値 | 早い（提供スピード） |
| **情緒価値** | 体験を通じて得られる感情的な価値 | うまい（食べる喜び・満足感） |
| **金銭価値** | 価格・オファーに関する価値 | 安い（コストパフォーマンス） |
| **信頼価値** | 実績・口コミ・権威に基づく信頼の価値 | 老舗・長年の実績 |
| **体制価値** | 流通・CS・販売体制がもたらすアクセスの価値 | どこにでもある（店舗数の多さ） |

### 5.3 提供便益とベネフィットシーン

価値軸で整理した後、その価値から**具体的なベネフィットシーン**を想定する。

| 項目 | 定義 | 例（疲労回復パジャマ × 肩こりターゲット） |
|---|---|---|
| **機能価値** | 商品の機能的な特徴 | 特殊繊維による血行促進 |
| **提供便益** | ユーザーが得られる具体的な価値 | 筋肉のこり改善・疲労回復の促進 |
| **便益提供シーン** | その便益が実感されるシーン | 朝起きた時に肩が軽い、寝起きのだるさがない |

**WHOごとにWHATは変わる**:
- 肩こりがひどい人 → 血行促進による筋肉のこり改善を訴求
- 寝違えが多い人 → 寝返りサポート機能を訴求
- ガジェット好き → 特殊繊維の技術的な新しさを訴求

## 6. WHY（訴求が刺さる理由の定義）

### 6.1 WHYの構造

WHYは「なぜこのターゲットにこの訴求が刺さるのか」を明確化する。
核となるのは、**そのターゲットだからこそ抱える具体的なペインか、強い願望**である。

```
便益のない現状シーン → 強いペイン / 強い願望 → だからこの訴求が刺さる
```

### 6.2 便益のない現状と強いペイン/願望

| 項目 | 定義 | 例（疲労回復パジャマ × 40代主婦） |
|---|---|---|
| **便益のない現状** | その便益がない生活でどんなシーンが起きているか | 毎朝身体が重く、家事を始めるのに時間がかかる |
| **強いペイン** | そのターゲットだからこそ感じる具体的な苦痛 | 疲れが取れない→子供の世話で休めない→自分の時間が一切ない悪循環 |
| **強い願望** | そのターゲットだからこそ持つ具体的な理想 | 朝すっきり起きて、余裕を持って1日を始めたい |

> **ポイント**: ペイン/願望は**そのターゲットだからこそ起きる具体的なもの**であるほど刺さる。汎用的な「疲れを取りたい」ではなく、「子育てで自分の回復に時間を割けない40代主婦だからこそ、寝るだけで回復できることに価値がある」のように、WHOと紐づいた具体性が重要。

### 6.3 欲求ファネル

ターゲットが課題に対してどの認知段階にいるかを4つの層で定義する。

| ファネル層 | 定義 | 例（疲労回復パジャマ） |
|---|---|---|
| **潜在層** | 課題をまだ捉えていない | 疲れは年齢のせいだと思い、改善しようとしていない |
| **準顕在層** | 課題はなんとなく感じているが、認識しようとしていない | 「最近疲れやすいな」と思うが、具体的に調べたりはしない |
| **顕在層** | 課題として認識し、向き合っている | 疲労回復の方法を積極的に調べている |
| **比較検討層** | 具体的な商品・サービスを使っていたり、検討している | サプリ、マッサージ機、整体などを既に試している/比較中 |

> **注意**: ファネル層の特定はWHOの認知状況とWHYのペイン整理を**行き来しながら**確定する。

### 6.4 情報障壁・行動障壁

ターゲットのファネル層に合わせて、なぜ課題が解決できていないかを整理する。

| 障壁タイプ | 定義 | 例（準顕在層の40代主婦） |
|---|---|---|
| **情報障壁** | 解決策にたどり着けない情報面の障壁 | パジャマで疲労回復できるという情報に接触していない |
| **行動障壁** | 解決策を知っていても行動できない障壁 | マッサージに通うのは時間もお金もかかるので後回しにしている |

### 6.5 想定競合

障壁の整理から、ターゲットが想起する**すべての解決策**を想定競合として定義する。
同カテゴリーの直接競合だけでなく、**代替手段も含む**。

| 競合タイプ | 定義 | 例（疲労回復パジャマ） |
|---|---|---|
| **直接競合** | 同カテゴリーの競合商品 | 他社の機能性パジャマ、リカバリーウェア |
| **代替手段** | 同じ課題を別の方法で解決する手段 | マッサージ機、整体・整骨院、サプリメント、入浴剤 |
| **無行動** | 何もしない（現状維持） | 「疲れは仕方ない」と諦めて我慢する |

> **ポイント**: ターゲットのファネル層によって想定競合は変わる。潜在層は「無行動」が最大の競合、比較検討層は直接競合が中心となる。

## 7. 思考プロセス：行き来しながら精度を高める

### 7.1 訴求開発は線形プロセスではない

コンセプト→WHO→WHAT→WHYと一方向に進めるのではなく、**各要素を行き来しながら**精度を高めていく。

```
        コンセプト（便益×カテゴリー）
          ↕          ↕
    WHO（ターゲット）  ←→  WHAT（訴求内容）
          ↕          ↕
        WHY（刺さる理由）
          ↓
    想定競合 → 差別化軸
```

### 7.2 典型的な行き来のパターン

1. **WHATの整理中にカテゴリー再定義が起き、コンセプトが変わる**
   - 例: 水の商品特徴を整理 → 保湿成分の価値に気づく → 「飲む化粧水」というカテゴリー再定義 → コンセプトが固まる

2. **WHYのペイン深掘り中にWHOの解像度が上がる**
   - 例: 「疲れが取れない」というペインを深掘り → 「子育て中で自分の回復に時間を割けない」と具体化 → WHOの制約条件が明確になる

3. **想定競合の整理中にWHATの差別化軸が変わる**
   - 例: マッサージ機が競合だと認識 → 「能動的に使う必要がない」が差別化軸 → 「寝るだけで」を訴求の核に追加

4. **欲求ファネルの特定中にWHOの認知状況が修正される**
   - 例: 準顕在層だと仮定 → ペインを掘ると実は課題認識がない潜在層だった → WHOの認知状況を修正

### 7.3 完成の判断基準

以下がすべて一貫して整理されていれば、訴求開発は完成。

- [ ] コンセプトが「便益×カテゴリー」で明確に定義されている
- [ ] コンセプトが世の中にない新しい便益として、イメージの広がる言葉になっている
- [ ] WHOが状態・認知・制約の3軸で具体的に定義されている
- [ ] WHATが5つの価値軸で整理され、便益シーンが想定されている
- [ ] WHYに強いペインまたは強い願望が、WHOに紐づいた具体性を持って定義されている
- [ ] 欲求ファネルの層が特定されている
- [ ] 想定競合（直接競合+代替手段）が整理されている
- [ ] 差別化軸が明確になっている
- [ ] コンセプト・WHO・WHAT・WHYの間に矛盾がない

## 8. 実践例：債務整理案件での適用

以下は、実際に債務整理案件で訴求開発を行った際の思考プロセスの要約。

### コンセプト

```
便益: 利息をやめられる → カテゴリー: 乗り換え
= 「利息の乗り換え」「利息断ち」
```

従来の「債務整理」というカテゴリーでは行動障壁が高すぎるため、「乗り換え」という既知のカテゴリーで再定義。

### WHO

| 軸 | 内容 |
|---|---|
| 状態 | まだギリギリ返せているが毎月キツい。口座残高は管理できていない |
| 認知 | 自分を「借金で困っている人」と認識していない。正常性バイアスで先送り |
| 制約 | 「債務整理」という言葉に心理的ハードルがある。弁護士に相談する行動が重い |

### WHAT

| 価値軸 | 内容 |
|---|---|
| 機能価値 | 利息カット・減額・返済期間の調整 |
| 金銭価値 | 200万借りて300万返す利息が0になる |
| 信頼価値 | 弁護士・司法書士による公的な手続き |

- **カテゴリー再定義**: 債務整理 → 利息の乗り換え
- **入口体験の分離**: 売るもの（債務整理）≠ 入口の体験（診断）

### WHY

| 項目 | 内容 |
|---|---|
| 便益のない現状 | 真面目に返し続けているが、利息分で元本が減らない |
| 強いペイン | 「返せている」という自己認識が行動を止めている。返し続けるほど損をしている |
| 欲求ファネル | 準顕在層（キツいのは薄々わかっているが、課題として向き合おうとしていない） |
| 情報障壁 | 「まだ返せている自分」は債務整理の対象外だと思っている |
| 行動障壁 | 「債務整理」の言葉が重い。弁護士に相談するハードルが高い |
| 想定競合 | おまとめローン（代替手段）、何もしない（現状維持）、他の債務整理広告（直接競合） |

### 差別化軸

既存広告は「借金がきつくて困っている人」（顕在層）を狙っているが、ボリュームゾーンは「まだギリギリ返せている人」（準顕在層）。この認知のズレ自体が差別化の起点。

> 「返済を続けている人ほど損をしている」という逆説をコンセプトの核に据えた。

## 9. AI活用時の指針

### 9.1 AIが得意なこと

- リサーチデータの収集・整理・構造化
- 既存広告の訴求パターンの分類
- 規制動向のリサーチ
- 条件を与えた上でのコンセプト案の大量生成
- WHOの複数パターン出し
- 5つの価値軸での整理
- ペイン/願望の洗い出し
- 想定競合のリストアップ

### 9.2 人間の判断が不可欠なこと

- どのインサイトが本質的かの選別
- コンセプトの言葉としての強度の判定（説明として正しいが言葉として弱い、の判断）
- ターゲットの微妙な心理状態の言語化
- 最終的なコンセプトの「筋の良さ」の判定
- 行き来の中でどこに立ち戻るべきかの判断

### 9.3 AIとの協業フロー

```
[AI] リサーチデータの構造化
 ↓
[AI] コンセプト候補の大量生成 + WHO/WHAT/WHYの叩き台
 ↓
[人間] 筋の良いコンセプト候補の選別
 ↓
[AI + 人間] 選別したコンセプトに対してWHO/WHAT/WHYを行き来しながら精度向上
 ↓
[人間] 最終的なコンセプト・訴求の確定
 ↓
[AI] 確定した訴求に基づく広告企画・台本生成（次フェーズ）
```
