# コンテンツ開発

## INPUT
- セールス設計図（知覚価値×知覚コスト、各要素に素材紐づけ済み）

## OUTPUT
- 表現手法の決定
- フック型の決定
- コンテンツとセールスのバランス設計
- フックリメイクの場合: 新フックのアプローチ

---

## 表現手法の選択

表現手法 = コンテンツとセールスのバランスの選択でもある:
- アニメ会話劇: コンテンツ強め（対話で引っ張り、会話にセールスが溶ける）
- BA比較: セールス強め（BA映像がセールスでありコンテンツ）
- UGC体験談: 混在（ストーリーがコンテンツ、中身がセールス）
- テロップ主体: セールス強め

N1のコンテンツ嗜好が選択基準。

## フック型の選択

- N1のコンテンツ嗜好がフック型を決める
- ファネルによって効く型が変わる
- 17型は `marketing_knowledge_base.md` を参照
- 型は構造。表現は型の中で自由

## コンテンツとセールスの関係

分離しない。同じ時間軸上で混ざり合う:

| パターン | 例 |
|---------|---|
| セールス自体がコンテンツ | 認知転換の内容自体が面白い/驚きがある |
| コンテンツで文脈→セールス | 先に文脈を作ってからセールスの核を出す |
| コンテンツにセールスが溶けてる | 体験談の中で自然に価値が伝わる |

## フックリメイク

### 定義
勝ちCRのフックだけ変えて、新規層を獲得する。ボディは触らない。

### 絶対制約
- ボディでCVするユーザーを連れてくる
- ボディとの接続（つなぎ）まで設計する
- 壁1a（止める）だけでなく壁1b（橋渡し）まで

### CVしなかった人の2パターン
1. フックとボディのズレで離脱 → 「何を伝えるか」をずらす
2. そもそもフックに反応しない → 「表現/切り口」を変える

### 思考手順
1. ボディの構造分解
2. CVする人のプロファイリング
3. 刺さりポイント特定
4. 取りこぼし層の仮説
5. 自分ごと化装置の設計
6. 別のフック型でボディに接続するフック案
7. 新フックで来る人がCVする人と一致するか検証

## フック生成の基本ルール

- **フックの本質は文脈づくり。** コンテンツ機能とセールス機能を同時に果たす
- 出来事を描写する。感情は書かない
- N1の脳内マップ上の言葉で入る。商品カテゴリーで入るな
- フックに詰められるものは全部詰める。0.5秒で判断される
- 既存ヒットと距離を作る（Andromeda対策）
- 「誰に言われるか」の解像度
- 自慢×自虐のバランス。LLMは自慢に寄りがち
- 短く。S0+S1で3秒以内
