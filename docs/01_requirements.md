# 売れるBUZZ AI - 要件定義書

## 1. プロジェクト概要

### 1.1 プロダクト名
売れるBUZZ AI（AX BUZZ）

### 1.2 プロダクトビジョン
コンセプトを入力するだけで、ショート動画広告の企画・制作・配信・分析・改善までを完全自動で回すAIエージェントシステム。

### 1.3 コンセプトの定義
「コンセプト」= 便益 × 商品カテゴリー の掛け算で定義される、ターゲットが一言聞いただけで欲しいと思う言葉。
- 例：どこでもドア = 「どこでも行ける」× 「ドア」
- 例：タケコプター = 「竹とんぼサイズの」× 「ヘリコプター」

### 1.4 ビジネスモデル
| 項目 | 内容 |
|---|---|
| MVP対象 | 10社 |
| 制作単価 | 動画1本 5万円 |
| 月額利用料 | 10万円〜30万円（プラン別） |
| 1社あたり制作量 | 週3企画 × 3フックバリエーション = 9本/週（MAX月20企画） |
| 対象媒体 | Meta / TikTok / YouTube / LINE / X |

### 1.5 組織体制
| 役割 | 担当業務 |
|---|---|
| アカウントプランナー（AP） | 営業・案件セットアップ・検証仮説設定 |
| メディアプランナー（MP） | 運用・レポーティング |
| クリエイティブプランナー（CP） | リサーチ・訴求開発・広告企画・台本制作 |
| クリエイティブディレクター（CD） | 編集概要・撮影・品質管理 |
| 編集者 | 動画編集（将来的にAI完全自動化） |

## 2. システム全体像

### 2.1 ワークフロー（10フェーズ）

```
[コンセプト入力]
    ↓
[Phase 1] 案件セットアップ ─── AP入力 → 承認
    ↓
[Phase 2] リサーチ ─── AI自動実行 → 承認
    ↓
[Phase 3] 訴求開発 ─── AI提案 → 人間選択 → 承認
    ↓
[Phase 4] 広告企画 ─── AI自動生成 → 承認
    ↓
[Phase 5] 台本制作 ─── AI自動生成 → 承認
    ↓
[Phase 6] 編集概要 ─── AI自動生成（素材選定含む） → 承認
    ↓
[Phase 7] 動画編集 ─── Remotion自動編集 → 承認
    ↓
[Phase 8] 入稿 ─── 動画名入力 → 自動入稿 → Slack通知 → 承認
    ↓
[Phase 9] 運用 ─── 5分間隔自動判断 → 承認（初期は人間確認）
    ↓
[Phase 10] レポーティング ─── 自動生成 → 人間確認 → 次サイクルへ
```

### 2.2 承認フロー
各フェーズに「承認ポイント」を設置。Notion上の承認ボタンで担当者が確認・承認する。
- 担当者は案件セットアップ時に事前設定
- 承認後に次フェーズが自動起動
- 将来的にはAIセルフフィードバックで承認不要化を目指す

### Phase 2: リサーチ

1. Web検索（自動）
2. 広告ライブラリ（Meta/TikTok）（自動）
3. クライアント提供資料（手動アップロード→AI解析）
4. SNS情報（自動）
5. 動画広告分析Pro（API連携）

#### リサーチ項目（リサーチシートの構造に準拠）

**商品理解**
| カテゴリ | 項目 |
|---|---|
| 基本情報 | 商品名、商品カテゴリー、情報ソース、製品体験メモ、前年度売上 |
| 機能価値 | 機能/成分/サービス、効果効能、研究データ、使用感、安全性 |
| 情緒価値 | デザイン（ベース/メイン/アクセント/テキストカラー、カラーイメージ）、体感（ストーリー、ブランドイメージ） |
| 金銭価値 | 通常価格、オファー、限定性 |
| 信頼価値 | 口コミ/レビュー/UGC、受賞歴、有名人、売上/企業実績、メディア掲載実績 |
| 体制価値 | 流通体制、CS体制、販売体制 |

**セグメント分析候補**
USP、機能価値、提供便益、便益のない現実、想定解決策、情報/行動障壁、セグメント分析候補

**市場/競合調査**
商品市場、代替市場（ソース/項目/割合 × 複数セグメント）

**既存顧客調査**
年齢、性別、職業、年収、世帯、購入目的、最重要判断軸、課題/願望、比較対象、購入阻害要因（各ソース/項目/割合）

**N1インタビュー**（将来的にAIアバターで自動化）
- 前提情報: 対象条件、URL、性別、年代、年収帯、居住地、職種、家族構成、ライフスタイル、メディア
- インタビュー: 商品関連の生活状況、一番の悩み/願望、理想の状態、欲求の種類、悩み/願望の期間、きっかけシーン、体感、背景情報、感情、想起した解決策、行動実績、カテゴリー認識、判断軸、比較商品、比較商品の認識、行動障壁/不安・リスク、新発見メモ

### Phase 4: 広告企画

#### 概要
- 決定された訴求に対して、興味の型 × 構成の型で広告企画を自動生成
- 1企画 = 1台本 → 3フックバリエーション = 3動画

#### 企画の構造（企画開発シートに準拠）
| セクション | 項目 |
|---|---|
| 前提情報 | 広告形式、広告種別、配信媒体、目標指標、目標数値、遷移先記事URL、遷移先LPURL |
| 広告企画 | コンテンツ、**興味の型**、**構成の型**（動画のみ）、FV（テキスト）、FV（映像）、構成 |
| 企画仮説 | 見る理由（見ない壁）、自分ごと化する理由（自分ごと化の壁）、信じる理由（信じない壁）、今すぐ行動する理由（行動しない壁） |

#### 興味の型（フックのタイプ）
| 型名 | 概要 |
|---|---|
| 商品興味 | 商品自体への興味を喚起 |
| エピソード興味 | ストーリー・体験談で引き込む |
| 手法興味 | 方法・やり方への興味 |
| 恐怖興味 | 不安・恐怖心を喚起 |
| 未来興味 | 理想の未来像で引き込む |
| 損失興味 | 損失回避心理を活用 |
| 常識破壊 | 常識を覆す切り口 |
| ターゲット指定 | 特定ターゲットを直接呼びかけ |

#### 構成の型（動画全体のフォーマット）
| 型名 | 概要 |
|---|---|
| UGC | ユーザー生成コンテンツ風 |
| アニメ | アニメーション形式 |
| 企業 | 企業公式風 |
| 語り | ナレーション主体 |
| 店舗体験 | 実店舗体験レポート風 |
| ドラマ | ストーリー仕立て |
| バナー動画 | バナー広告の動画版 |
| イラスト | イラスト主体 |
| インフルエンサー | インフルエンサー起用 |
| 雑学 | 豆知識・雑学形式 |
| AI | AI生成コンテンツ |

### Phase 5: 台本制作

#### 概要
- AIが企画に基づいて自動生成
- 基本尺：60秒以内
- テキストベースの台本

#### 台本構造（4つの壁を超える構成）
```
[フック] ── 見ない壁を超える（興味の型に基づく）
    ↓
[関心/自分ごと化] ── 自分ごと化の壁を超える（悩み/常識破壊/自分ごと化トピック）
    ↓
[コンセプト提示] ── コンセプトバーン
    ↓
[商品紹介] ── コンセプトが伝わる商品紹介
    ↓
[ベネフィット] ── 信じない壁を超える
    ↓
[オファー] ── 金銭的価値の提示
    ↓
[CTA] ── 行動しない壁を超える（今すぐ行動する理由）
```

※構成の型によって構造は変動する（ドラマ型、アニメ型等で異なる）

#### フックバリエーション
- 1企画に対して3パターンのフックを生成
- 同一訴求で切り口を変える（例：商品興味 / エピソード興味 / 手法興味）

### Phase 6: 編集概要（MVP最優先）

#### 概要
- 台本テキストに基づいて編集指示を自動生成
- 素材の自動選定・自動紐付け
- **現在最も時間がかかっている工程 → 最優先で自動化**

#### 編集概要の項目（動画名シートに準拠）
| カテゴリ | 項目 | 入力方式（現状） | 自動化後 |
|---|---|---|---|
| 基本 | 全体文字数 | 自動計算 | AI自動 |
| 台本 | 台本テキスト（セリフ/ナレーション） | 手動 | AI自動 |
| 音声 | 音声素材 / 注釈 | 手動 | AI自動 |
| レギュ | レギュレーションチェック欄 | 手動確認 | AI自動チェック |
| BGM | 素材名 / 素材スクショ / 素材URL | 手動 | AI自動選定 |
| 映像 | カット（素材指定） | **手動（最大ボトルネック）** | AI自動選定 |
| テロップ | テロップ内容・スタイル | 手動 | AI自動 |
| エフェクト | モーションエフェクト | 任意手動 | AI自動 |
| 効果音 | SE / ME | 任意手動 | AI自動 |
| その他 | その他編集指示 | 任意手動 | AI自動 |
| 修正 | 修正依頼①② / 備考 | 手動 | フィードバックループ |
| 参考 | 参考動画/テロップ | 手動 | AI提案 |

#### 素材リストとの連携
| 項目 | 詳細 |
|---|---|
| 素材形式 | 動画/画像/イラスト等（ドロップダウン） |
| 素材ジャンル | カテゴリ分類（ドロップダウン） |
| 素材名 | 一意の識別名 |
| 素材概要 | 内容説明 |
| 登場人物 | 出演者情報 |
| ステータス | 利用可/利用禁止/クライアント提供等 |
| URL | Google Drive等のリンク |
| カット指定 | 使用するカット範囲 |

#### 素材の取扱いルール
- クライアント提供素材は当該案件のみ使用可
- 利用禁止素材は全案件で使用不可
- 汎用素材は案件横断で利用可
- マルチテナント：A社素材はB社に表示されない

### Phase 7: 動画編集

#### 技術
- **Remotion**（React製動画生成フレームワーク）を使用
- 編集概要（Phase 6）のデータをRemotionコードに変換して自動編集

#### 自動化レベル
1. **Phase 1（MVP）**: 編集概要を元にRemotionで自動生成 → 人間が微調整
2. **Phase 2**: AIセルフフィードバックで自動改善
3. **Phase 3**: 完全自動（人間の調整不要）

#### 調整可能パラメータ
- 素材（カット割り・タイミング）
- テンポ
- テロップスタイル
- BGM
- モーションエフェクト
- サウンドエフェクト
- 音声（声優録音 or TTS）

#### 音声
- 声優録音：手動アップロード
- TTS：API経由で自動生成（サービス未定、各種対応）

#### 要件
- **入力**: 動画名のみ
- **出力**: 各媒体への自動入稿

#### 対応媒体と入稿内容
| 媒体 | API | 入稿内容 |
|---|---|---|
| Meta | Meta Ads API（接続済） | キャンペーン/広告セット/広告/クリエイティブ |
| TikTok | TikTok Marketing API（接続済） | キャンペーン/広告グループ/広告 |
| YouTube | Google Ads API | キャンペーン/広告グループ/広告 |
| LINE | LINE Ads API | TBD |
| X | X Ads API | TBD |

#### 入稿フロー
1. 動画名を入力
2. 事前に整理された入稿設定（案件×媒体×キャンペーン構造）を参照
3. 各媒体APIで自動入稿
4. Slack通知で入稿完了を通知
5. 入稿内容のログをNotionに記録

### Phase 9: 運用

#### 自動運用ルール（案件ごとに設定）
| 判断 | 条件例 |
|---|---|
| **停止** | CPA×1でCVが0 |
| **継続** | ROI 100% |
| **拡大** | ROI 150%以上 |
| **縮小** | 拡大期の縮小条件に準拠 |

#### 実行頻度
- 5分間隔で自動判断
- 初期は人間確認を挟む → 段階的に完全自動化

#### データ取得
- 各媒体APIからリアルタイムでパフォーマンスデータ取得
- CV数、COST、CPA、CTR、CVR等

### Phase 10: レポーティング

#### レポート種別
| レポート | 頻度 | 内容 |
|---|---|---|
| 日次レポ | 毎日 | 日ごとの推移（CV/COST/CPA × gross/net × 全体/媒体別） |
| 週次レポ | 毎週 | 週間パフォーマンスサマリー |
| 月次レポ | 毎月 | 月間パフォーマンスサマリー + グラフ |
| CRレポ | 随時 | クリエイティブ単位の効果（日付指定） |
| CRランキング | 随時 | 案件別：CV数順、案件横断：消化金額順 |
| 検証レポ | 週次 | 事前仮説 × 検証結果 → ナレッジ蓄積 |
| サマリー | リアルタイム | 本日/昨日/週次/月次 × CV/COST/CPA |

#### マクロ分析
1. コンセプト別の成果
2. 訴求別の成果
3. 訴求 × 構成の型別の成果
→ 最も成果期待値の高い展開軸をレポーティング

#### ミクロ分析
1. 2秒視聴率の高いフックベクトル（興味の型別評価）
2. 各構成軸（話者設定/状況設定/エピソード/ペイン/商品紹介/ベネフィット/オファーCTA）→ CTR軸・CVR軸での評価
3. 編集軸（カット割/テロップ/BGM/セリフ音声/SE/ME）→ CPA軸での評価
4. 視聴維持率から離脱箇所の特定

#### 提出先
- 社内用 / クライアント提出用（フォーマットは一旦同一）

## 4. AIチャット機能

### 概要
自然言語で質問すると、広告データ・ナレッジ・レギュレーション等を横断的に検索・回答するチャットインターフェース。

### 展開ロードマップ
| フェーズ | UI | 対象ユーザー |
|---|---|---|
| MVP | Notion上のAIチャット | 社内メンバー |
| Phase 2 | Slack Bot | 社内メンバー |
| SaaS化 | 独自UIチャットインターフェース | 社内 + クライアント |

### 機能
1. **配信結果の確認**: 「この案件のCPAは？」「今週のCV数は？」
2. **訴求開発・広告企画の相談**: 「この訴求の改善案は？」
3. **過去ナレッジの確認**: 「前回効果が高かったフックは？」
4. **レギュレーション確認**: 「この表現は使える？」

## 9. 非機能要件

### 8.1 マルチテナント
- MVP段階からクライアント間のデータ完全分離
- A社の素材/ナレッジ/レポートはB社からアクセス不可
- 汎用素材のみ横断利用可

### 8.2 データ永続化
- Notion（MVP段階のデータ管理）
- Supabase（ベクトル検索・ナレッジ蓄積用、検討中）
- Google Drive（素材格納）

### 8.3 通知
- Slack通知（入稿完了、承認依頼、ステータス変更等）

### 8.4 セキュリティ
- クライアント提供素材の利用制限管理
- 利用禁止素材のフラグ管理
- API認証情報の安全な管理

### 8.5 SaaS化準備
- MVP: Notion上（自社用）
- SaaS化: 独自UI（クライアント自身が操作）
- 自社用とクライアント用でUIを切り分け
