対話を通じて継続的に更新する。

## Lv1: マーケティング論

### マーケティングの本質
**マーケティングとは需要と供給を繋ぐこと。**

### ビジネスの基本構造
- ビジネス = 需要に対して供給を届け、報酬をもらうこと
- 人は**感情**を求めている。得られるプラスの感情 = ベネフィット
- 感情を言語化することは難しい → **シーンで切り取る**必要がある
- 製品（原材料を加工したもの）に情報を付加したものが商品。マーケターの守備範囲は**商品を作ること**

### 3つの本質的な問い
1. **どの需要でNo.1を取るのか？**（ターゲティング）
2. **どのような供給を提案するのか？**（コンセプト）
3. **どのようにコミュニケーションするのか？**

### 需要の捉え方
- 欲求 = 何かを欲する心。需要 = 商品・サービスを求める欲求
- 同じ商品でも色んな需要が存在する。お金を払う金額も競合も違う
- 需要ベースでSTP分析を行う（人ベースではなく）
- 青い需要（競合少・人数多・購買意欲額高）と赤い需要（競合多・人数少・購買意欲額低）
- 泳ぐ需要 = 戦うことを決めた青い需要

### コンセプト
- コンセプト = **便益 × カテゴリー**の掛け算で生まれる、世の中にない新しい便益を表す言葉
- カテゴリーの再定義が鍵（水→飲む化粧水、債務整理→利息の乗り換え）
- 良いコンセプト: 一言で「欲しい」「知りたい」と思える。イメージが広がる
- コンセプトは設計の起点であると同時に、視聴者の脳内に植え付ける新しい概念の表現物
- マーケティングは情報戦。消費者の脳内に情報を一番乗りで残せた者が勝つ。コンセプトを最初に植え付けた者がカテゴリーオーナーになる

### 訴求開発フレームワーク（WHO / WHAT / WHY）

**WHO（ターゲット定義）: 3軸**
| 軸 | 定義 |
|---|------|
| 状態 | どんな生活・身体・心理状態にある人か |
| 認知状況 | その課題に対してどう認識しているか |
| 制約条件 | 行動を阻む具体的な制約は何か |

一つのコンセプトに対して複数のWHOが存在する。

**WHAT（訴求内容）: 5つの価値軸**
| 価値軸 | 定義 |
|---|------|
| 機能価値 | 商品の機能・性能がもたらす具体的な価値 |
| 情緒価値 | 体験を通じて得られる感情的な価値 |
| 金銭価値 | 価格・オファーに関する価値 |
| 信頼価値 | 実績・口コミ・権威に基づく信頼の価値 |
| 体制価値 | 流通・CS・販売体制がもたらすアクセスの価値 |

WHOごとにWHATは変わる。価値軸→提供便益→便益提供シーンの順で具体化。

**WHY（刺さる理由）:**
- 便益のない現状シーン → 強いペイン / 強い願望 → だからこの訴求が刺さる
- ペイン/願望は**そのターゲットだからこそ**の具体性が重要

**欲求ファネル:**
| 層 | 定義 |
|---|------|
| 潜在層 | 課題をまだ捉えていない |
| 準顕在層 | なんとなく感じているが向き合っていない |
| 顕在層 | 課題として認識し向き合っている |
| 比較検討層 | 具体的な商品を検討/使用中 |

**障壁:**
- 情報障壁: 解決策にたどり着けない
- 行動障壁: 知っていても行動できない

**想定競合:** 直接競合 + 代替手段 + 無行動（何もしない）

### 訴求開発は線形プロセスではない
コンセプト↔WHO↔WHAT↔WHYを**行き来しながら**精度を高める。

### ポジショニング
- ポジショニング = ユーザーの脳内マップ上のどこに自社商品を置くか
- **一番乗り以外に価値はない。2番目は忘れ去られる**
- ポジショニングマップは自社都合で作るものではない。ユーザーの脳内に既にマップが存在している
- 手順: ①ユーザーの脳内マップを知る（リサーチ） → ②空いてるポジションを見つける（競合×顧客認知） → ③一番乗りする（コンセプト→セールス）
- N1の「解決策 - 知っていることと印象」がユーザーの脳内マップそのもの

### セールス論
**セールスの成立条件: 知覚価値 > 知覚コスト**

顧客が「これは自分にとって最善の選択肢だ」と事実に基づいて判断した時にセールスが成立する。
「買わせる」「売りつける」「大きく見せる」はNG。価値は事実から生まれる。顧客と商品に愛を持つ。

#### 知覚価値を上げる（事実ベース）
| 要素 | 問い |
|------|------|
| 便益の正しい解釈 | 事実からどんな価値が導かれるか |
| ポジショニング | ユーザーの脳内マップのどこに一番乗りするか |
| 唯一性 | 他の選択肢ではなぜこの便益が得られないか |
| 自分ごと化 | なぜこの人に関係あると思えるか |
| 証拠 | 事実として信じられる根拠は何か |
| 便益の重なり | 複数の価値が同時に手に入るか |

#### 知覚コストを下げる
| 要素 | 問い |
|------|------|
| 金銭コスト | 何と比較して適正か（アンカリング） |
| 金銭の正当性 | なぜこの価格で可能か（安かろう悪かろうの払拭） |
| リスクコスト | 事実として何がリスク0か |
| 比較コスト | 考えなくていい状態を作れるか（判断基準を提示） |
| 手間コスト | 行動の負荷はどれだけ小さいか |
| 心理コスト | 最初の一歩のハードルは何か |
| 時間コスト | どれくらいで結果が出るか |

### USPとオファー
- USP = 商品コンセプトをオファー化したもの = セールスのコンセプト
- オファー = 提案する条件
- 良いオファー: ①お得感 ②リスクが少ない ③今すぐの理由 ④わかりやすい
- マーケティング > セールス > オファー の階層

### マーケティングは情報戦
人々の頭の中にどのような情報を残すか、どのような情報を提供して取引を成立させるかは全て情報の取り扱い方で変わる。

### リサーチ
マーケティングはリサーチで8割決まる。

**3C分析（広告制作における）:**

| C | 整理すること |
|---|------------|
| **Customer（顧客）** | どんな情報を持っているか。どんな状態か。何を知っていて何を知らないか。どんな感情・欲求を抱えているか |
| **Competitor（競合）** | 何を言っているか。どんな訴求をしているか。どのファネル層を狙っているか |
| **Company（自社商品）** | 何が言えるか。特徴・効果・証拠・ベネフィットの整理 |

**マーケット知識:**
- そのマーケットそのものの専門知識が極めて重要
- 例: 「矯正でフェイスラインまで変わる」という事実を知っているかどうかで、訴求の幅が全く変わる
- マーケットの境界を再定義する視点が新しいコンセプトを生む（矯正マーケット→美容医療マーケットへの拡大）
- カテゴリー再定義の源泉はマーケット知識にある

**商品分析がコンセプトの質を決める:**
- コンセプトは発想力ではなく商品理解の解像度で決まる
- 商品の機能を分解するほど便益の切り口が増え、競合が言っていない新しいコンセプトを見つけられる
- Company分析の解像度が低いと、全ての下流工程（WHAT→コンセプト→セールス→コンテンツ）の天井が下がる

**3つの目:**
- 鳥の目（市場全体）
- 虫の目（N1の深掘り）
- 魚の目（流れ＝変化＝新しい需要）
- すべて需要の観点で捉える。競合は鳥の目ではなく虫の目から出る

## Lv2: 広告論

### 広告の位置づけ
コミュニケーション = 情報を伝達して取引を促すこと。広告はコミュニケーションの手段。

### 広告コンテンツの分析原則
- テキストではなく**人間を分析する**
- 「何が書いてあるか」ではなく「このコンテンツでCVする人間は誰か」を特定する
- 教育パートの有無で対象者のファネル位置が決まる

### CVする人のプロファイリング（6軸）
| 軸 | 問い |
|---|------|
| 悩み・理想 | 何に悩み、何を知っていて何を知らないか？ |
| 解決方法 | どんな解決策を知っていて、どれを検討中か？ |
| 重視する基準 | 何で選ぶか？ |
| 過去の行動 | 何を試して、どうだったか？ |
| 障壁 | 踏み出せてない理由は？ |
| コンテンツ嗜好 | どんなコンテンツをつい見てしまうか？ |

### ボディ分析の手順
1. 構造分解: フック / 教育 / 信頼構築 / オファー / CTA
2. CVする人のプロファイリング（6軸）
3. ボディの各パートが6軸のどこに対応するか特定
4. 教育パートあり → 認知転換が価値（潜在〜準顕在層向け）
5. 教育パートなし → 視聴者は既に知っている前提（顕在〜比較検討層向け）

### 4つの壁
| 壁 | 問い | 担当 | KPI |
|----|------|------|-----|
| 1a. 見始めない | なぜスクロールを止めるのか？ | フック（最初の0.5秒） | 3s視聴率 |
| 1b. 見続けない | なぜボディまで見るのか？ | フック→ボディのつなぎ。教育コンテンツの引き | 25%再生率 |
| 2. 信じない | なぜ本当だと信じ、自分に関係あると思うのか？ | ボディ（教育+信頼構築） | 完視率・CTR |
| 3. 行動しない | なぜ今やるのか？ | オファー+CTA | LPCVR・CVR |

### クリエイティブの横展開（リメイク）
- 本体（ボディ）の価値は実績で証明済み。変数は入口だけ
- 入口を変えて同じボディでCVする**別の人間**を連れてくる
- 取りこぼし2種: 入口で止まらなかった人 / 入口とボディのズレで離脱した人

## Lv3: 動画広告論

### フックとボディの役割分担
- フック = 壁1a（見始めない）を突破
- つなぎ = 壁1b（見続けない）を突破。教育コンテンツ自体が見続ける理由になりうる
- ボディ = 壁2（信じない）を突破
- CTA = 壁3（行動しない）を突破

### 17のフック型（案件非依存）
| # | 型 | 構造 |
|---|---|------|
| ① | ギャップ告白 | 現在の良い状態→過去の暴露 |
| ② | ゼロリスクBA | リスク0宣言→BA映像 |
| ③ | 執念の探求 | 欲求→大量比較→正解 |
| ④ | 第三者目撃 | 他者の変化を目撃→何したの？ |
| ⑤ | 上位互換 | 検討中の選択肢→こっちが良い |
| ⑥ | 衝撃会話 | 衝撃セリフ→理由判明 |
| ⑦ | 期限/緊急 | 期限→間に合う方法 |
| ⑧ | 共感悩み | 悩み描写→共感 |
| ⑨ | 権威ランキング | 権威性→教える/ランキング |
| ⑩ | リベンジ | 怒り→変身 |
| ⑪ | キャラ設定 | ペルソナ宣言→体験 |
| ⑫ | アニメ会話劇 | キャラ間会話で引っ張る |
| ⑬ | データ恐怖 | 統計/数字→恐怖 |
| ⑭ | オファー直球 | 商品/価格を直接提示 |
| ⑮ | ミスリード逆転 | ネガティブで止める→実はポジティブ |
| ⑯ | ターゲット名指し×危機宣告 | お前のこの状態→終わりの合図→今すぐ |
| ⑰ | 選出通知 | あなたが選ばれた→オファー |

ターゲット属性によって効く型の分布が変わるが、構造自体は普遍的。

### フックリメイクの思考手順
1. ボディの構造分解
2. CVする人のプロファイリング（6軸）
3. ボディの刺さりポイント特定（認知転換の有無）
4. 現フックの取りこぼし層の仮説
5. 自分ごと化装置の設計（一般論→「これ私のことだ」に変えるトリガー）
6. 別のフック型で、取りこぼし層を、ボディに自然に接続するフック案を生成
7. 新フックで来る人がボディでCVする人と一致するか検証

### フック生成のルール
- **出来事を描写する。感情は書かない。** 感情は視聴者が勝手に生成する
- 既存ヒットと**距離を作る**（Andromeda対策）。テンプレ展開はNG
- ヒットCRの「なぜ売れてるか」を**仮説分解→仮説別横展開**
- 「誰に言われるか」の解像度が重要
- LLMの役割: 型の提案・参考例提示・距離チェック。台本の最終仕上げは人間

### フック分析の方法
- ラベル（CR名）ではなく実テキスト（S0テロップ+S1テロップ+台本1文目）で分析
- 台本全文を使わない（ボディが混入する）
- 分類はLLMで（キーワードマッチは精度不足）

## Lv1→Lv3の接続: コンセプトと動画広告の関係

### ボディの認知転換 = コンセプトの伝達
- コンセプト（便益×カテゴリー再定義）を視聴者に伝えるのがボディの教育パート
- 教育パートがあるCR = **コンセプト伝達型** = 潜在〜準顕在層向け
- 教育パートがないCR = **USP/オファー提示型** = 顕在〜比較検討層向け（コンセプトは既知の前提）

### フックリメイクの余地
- コンセプト伝達型のボディ → フックリメイクの余地が大きい（コンセプトを知らない人がまだ大量にいるから。入口を変えれば別の層を連れてこれる）
- USP提示型のボディ → フックリメイクの余地が小さい（既にコンセプトを知っている顕在層しかボディが刺さらない）

### WHO/WHAT/WHYと4つの壁の対応
| 訴求開発 | 動画広告の壁 | 対応 |
|----------|------------|------|
| WHO（状態・認知・制約） | 壁1a（見始めない） | フックがWHOの「状態」「認知」にマッチする表現かどうか |
| WHAT（便益・カテゴリー） | 壁1b〜壁2（見続けない〜信じない） | ボディの教育パートがWHATのコンセプトを伝え、信頼構築が信憑性を担保 |
| WHY（ペイン・願望・障壁） | 壁2〜壁3（信じない〜行動しない） | ペイン/願望が「自分ごと化」を、障壁突破がオファー設計を駆動 |
| 欲求ファネル | ボディの教育パートの有無 | 潜在〜準顕在→教育必要、顕在〜比較検討→教育不要 |
| 想定競合 | フックの型選択 | ⑤上位互換は代替手段が想定競合、⑮ミスリード逆転は直接競合が前提 |

### 動画CR制作の全体フロー

```
リサーチ
  ↓
マーケティング戦略の立案（訴求開発）
  ↓
企画開発（ファネル × HOW）
  ↓
構成制作
  ↓
台本制作
  ↓
編集概要
  ↓
編集
  ↓
レギュレーションチェック
  ↓
fix
```

| フェーズ | 内容 | アウトプット |
|---------|------|------------|
| **リサーチ** | 3C分析 + マーケット知識。マーケティングの8割がここで決まる | リサーチデータ |
| **訴求開発** | WHO/WHAT/WHY + コンセプト（便益×カテゴリー） | 訴求開発シート |
| **企画開発** | ファネルのどの層に、どう伝えるか（HOW）。各パートの目的と大枠の内容を決める | 企画書 |
| **構成制作** | 何をどんな順番で伝えるか | 構成案 |
| **台本制作** | 材料（BA映像、症例数、価格等）と紐づけて具体化 + 一つ一つの言葉の強さを磨く | 台本 |
| **編集概要** | 編集者への指示書 | 編集概要書 |
| **編集** | 映像の制作・編集 | 動画素材 |
| **レギュレーションチェック** | 薬機法・景表法等の規制確認 | チェック結果 |
| **fix** | 最終確定 | 完成CR |

### 企画開発のHOWの粒度

HOW = ファネルのどの層に対して、以下の要素をどう設計するか

| 要素 | 内容 |
|------|------|
| **表現手法** | vlog / アニメ / 一人語り / 企業視点のお知らせ 等 |
| **構成** | 何をどんな順番で伝えるのか？ |
| **フック** | 冒頭で何をどう見せて止めるか？（17型） |
| **台本** | 具体的に何と言うか？ |

### 企画開発に必要な2つの視点

| 視点 | 役割 |
|------|------|
| **セールスライティング** | 行動させる。壁2（信じない）→壁3（行動しない）の突破。証拠・オファー・CTA |
| **コピーライティング** | 注意を引き、記憶に残す。壁1a（見始めない）→壁1b（見続けない）の突破。言葉の力 |
| **コンテンツメイキング** | 見続けたいと思わせる。広告ではなくコンテンツとして成立させる。壁1b→壁2をシームレスに |

セールスライティングだけだと売り込み臭で離脱する。コピーライティングだけだと記憶に残るが行動しない。コンテンツメイキングだけだとエンタメで終わりCVしない。**3つ全てが必要。**

### 今日のセッションの議論の位置づけ
- フック17型の分析 = **企画開発**（表現手法・フック）の知見
- フックリメイク = **企画開発〜構成制作**の手法（ボディは既存、フック+つなぎを再設計）
- ボディ分析 = **訴求開発の結果がどう企画に落ちているか**の検証

### 目的・内容・表現の3レイヤー
広告制作の思考は3つのレイヤーに分離して考える。

| レイヤー | 定義 | 動画広告での対応 |
|---------|------|----------------|
| **目的** | どの壁を突破するか | 壁1a/1b/2/3のどれを担当するパートか |
| **内容** | 何を伝えるか | コンセプト、USP、ペイン、オファー等 |
| **表現** | どう伝えるか | 17型のフック型、フォーマット（アニメ/UGC/テロップ主体等） |

内容を考えてから表現する。表現から入らない。

### 構成の核 = セールスの核 = 認知転換

どのファネル層向けでも、構成には必ず**認知転換**が存在する。これがセールスの核であり、構成の核。

| ファネル層 | 認知転換の対象 | 例（ローコスト） |
|-----------|-------------|----------------|
| 潜在〜準顕在 | 解決策の認知転換 | 「矯正でEラインが変わる」（知らなかった→そうなんだ） |
| 顕在〜比較検討 | 判断基準の認知転換 | 「選ぶ基準は仕上がりの精度」（価格で選んでた→精度で選ぶべきなんだ） |

認知転換が起きた後、全ての情報がその新しい軸で評価される。起きなければ同じ情報でも刺さらない。

### セールスの核 = セールスのフック

セールスの核は構成の真ん中ではなく**セールスの入口**。ここでグッと引きつけ、そこから知覚価値の最大化と知覚コストの最小化が始まる。

```
セールスの核（＝セールスのフック）で引きつける
  ↓
知覚価値の最大化（証拠/自分ごと化/便益の重なり/唯一性）
  ↓
知覚コストの最小化（金銭/リスク/比較/手間/心理/時間）
  ↓
CTA
```

**核の前に何を置くかはN1の認知状態で変わる:**
- コンセプト自体が強い → 核から始まる
- 核だけだと信じてもらえない → 権威性を先に出す（「NHKで放送された〜」）
- N1が潜在層 → 自分ごと化を先に出す（悩み描写→核）

毎回決まった順番ではない。

### コンテンツとセールスの関係

コンテンツとセールスは分離しない。同じ時間軸上で混ざり合う。

| パターン | 例 |
|---------|---|
| セールス自体がコンテンツ | 認知転換の内容自体が面白い/驚きがある |
| コンテンツで文脈→セールス | 先に文脈を作ってからセールスの核を出す |
| コンテンツの中にセールスが溶けてる | 体験談の中で自然に価値が伝わる |

表現手法の選択 = コンテンツとセールスのバランスの選択でもある:
- アニメ会話劇: コンテンツ強め（対話で引っ張り、会話にセールスが溶ける）
- BA比較: セールス強め（BA映像がセールスでありコンテンツ）
- UGC体験談: 混在（ストーリーがコンテンツ、中身がセールス）

各瞬間が「コンテンツとして見続ける理由」と「セールスとして知覚価値/コストを動かす機能」の両方を持っていることが重要。

### コンセプト開発の行き来

便益→N1→コンセプト表現は線形ではなく行き来する:
1. 商品の便益から「こんな人が欲しがりそう」と仮説を立てる
2. その人の解像度を上げる（N1定義）
3. N1の理解が深まるとコンセプトの言葉が変わる
4. 人の発見から入ることもある

### ショート動画広告の本質

射幸性の高いオーガニック動画の間に挟まる。その中でしっかり見られるものでないといけない。

**良い動画広告の条件:**

1. **セールスの核の魅力が最大になるコンテンツ設計**
   - コンテンツパートはセールスの核がもっとも魅力的に伝わるための前戯
   - ぶっ飛んだもので見せればいいわけではない。セールスの核に自然につながるコンテンツでないと意味がない

2. **ターゲットだけに見せ、ターゲットなら絶対に見る**
   - 無駄な人に広告を見せない（ターゲットフィルター）
   - その代わり、ターゲットなら絶対に止まる（ターゲット確度）

3. **コンテンツからセールスへの自然なブリッジ**
   - 途中で「うわ、広告か」とならない
   - あくまで自然に見続ける中で、セールスの核（認知転換）に到達する

### コンセプトワードの作り方

コンセプトワードは一語である必要はない。便益を重ねて「一言で聞いて欲しくなる」を達成する。

**良いコンセプトの条件:**
- 一言で便益が伝わる
- これまでにない新しいものだと思う
- ターゲットなら欲しくなる

**便益の重ね方:**
- 便益×便益×カテゴリーで「それ全部叶うの？」という贅沢感を作る
- 例: 「ダウンタイム0の骨格矯正」= リスク0便益 × 骨格レベルの変化 × 矯正カテゴリー
- 例: 「2ヶ月で顔面アプデ」= スピード便益 × 変化の大きさ便益
- 1語で全部伝えようとしない。複数便益の組み合わせで引力が生まれる

**コンセプトワードのNG:**
- 便益が伝わらない（美容矯正 → 何が良くなるか不明）
- カテゴリーが嘘になる（DT0整形 → 整形ではない）
- 新しさがない（歯並び矯正 → 既知）
- 説明的すぎる（顔が変わる矯正 → イメージが広がらない）

### コンテンツ分析の型

コンテンツの役割 = セールスの核が最大限刺さる文脈づくり

**Step 1: セールスの核（認知転換）を特定する**
- 視聴者の頭の中が変わる瞬間はどこか？

**Step 2: その認知転換が最大限刺さるには、直前にどんな認知状態が必要か？**
- 転換の直前に、視聴者の頭の中にどんな事実/感情/問いが存在していれば、インパクトが最大になるか？

**Step 3: コンテンツパートがその認知状態を作れているか確認する**
- 各要素がStep 2の認知状態構築に向かっているか？
- 余計な情報が入っていないか？

全ての要素がセールスの核に向かっている状態が、良い構成。

### 設計と視聴体験の方向は逆

```
設計:    コンセプト → セールス → コンテンツ
視聴者:  コンテンツ → セールス → コンセプト
```

設計で最初に決まるのはコンセプト（便益×カテゴリー）。そこからセールスの核（認知転換）が決まる。最後にそれを最も魅力的に伝えるコンテンツを設計する。コンテンツ（表現）から考え始めてはいけない。

### 言葉の強さ
- 言葉の強さ = **一つの言葉から伝わる情報量**
- 人間の記憶はネットワーク構造。一つの言葉に対して複数の意味・関連性で記憶している（活性化拡散モデル）
- 「マック」→ ポテト、ハンバーガー、黄色いロゴ、ハッピーセット… 一語で大量の情報が展開される
- ユーザーの脳という記憶装置を使って、伝わる情報量を増やす
- 動画広告は音声で伝えるから、**口に出して読んだ時の自然さ**も重要
- 同じことを伝えるなら**できる限り短い文章**が良い

### 制作の実務

**リサーチ:**
- 競合分析は動画分析PROから集める
- N1リサーチは全部やる（インタビュー + 口コミ + SNS）

**訴求開発:**
- 訴求は1案件につき3つくらい考える
- カテゴリー再定義のセンスは事例の大量インプットで磨く

**企画開発:**
- 同じ訴求でもファネル層ごとにCRを分けて作る
- 表現手法は複数並走させる

**体制・スピード:**
- 全工程を1人でやる
- 通常1-2週間 → 1日でできるようにしたい

## 勝ちCRからマーケティング戦略を逆算する方法

**売れてる動画 = 市場が答え合わせした結果。** 1本の勝ちCRから制作フローを逆順にたどることで、マーケティング戦略を丸裸にできる。

### 逆算の流れ

```
売れてる動画
  ↓ 逆算
台本: 何と言っているか（言葉の具体的内容）
  ↓ 逆算
構成: 何をどんな順番で伝えているか（パートの並び）
  ↓ 逆算
企画: どのファネル層に、どんなHOWで伝えているか（表現手法・フック型）
  ↓ 逆算
訴求: WHO / WHAT / WHY / コンセプト
  ↓ 逆算
リサーチ: どんな3C分析・マーケット知識がこの訴求を生んだのか
```

### 各段階での問い

| 段階 | 問い |
|------|------|
| 台本 | 具体的に何と言っているか？どんな素材（BA、数字、証言）を使っているか？ |
| 構成 | フック→教育→信頼構築→オファー→CTAのどれが入っていてどれが入っていないか？順番は？ |
| 企画 | 教育パートの有無でファネル層を特定。表現手法は何か？フックは17型のどれか？ |
| 訴求 | WHO: この広告でCVする人は誰か（6軸）。WHAT: 便益×カテゴリーは何か。WHY: なぜこの人にこの訴求が刺さるのか |
| リサーチ | この訴求を作るために必要だったマーケット知識は何か？3Cのどの情報がこの判断を可能にしたか？ |

### 逆算で得られるもの
- 競合の戦略の理解（どの需要を狙っているか）
- 自社で再現可能な訴求パターンの発見
- まだ誰も狙っていない需要の発見（競合の隙間）
- フックリメイクの精度向上（元CRの戦略を正確に理解した上でフックを変える）

---

### コンセプトの差別化がコンテンツ設計を変える（実例）

同じ商品（無料診断→債務整理）でもコンセプトが違えば全てが変わる:

| | 既存市場 | 新コンセプト |
|---|---------|-----------|
| コンセプト | 借金減額診断 | 利息カット診断 |
| 便益の受け取り | 借りたお金を踏み倒す（罪悪感） | 余計に取られてるものを止める（正当な権利） |
| コンテンツ設計 | 借金で苦しいエピソード | 利息の計算で損を可視化 |
| 行動障壁 | 高い（罪悪感） | 低い（正当な行為） |

コンセプトが起点。コンセプトが変わればセールスの核が変わり、コンテンツの設計が変わる。
