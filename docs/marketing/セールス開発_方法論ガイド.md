# セールス開発

## INPUT
- 訴求開発のアウトプット（コンセプト / N1 / WHO×WHAT×WHY）

## OUTPUT
- 知覚価値×知覚コストの設計図（各要素に具体的な素材を紐づけた状態）

---

## 前提

- セールスの成立 = **知覚価値 > 知覚コスト**
- 「買わせる」「大きく見せる」はNG。価値は事実から生まれる。顧客と商品に愛を持つ

## 知覚価値を上げる（事実ベース）

| 要素 | 問い | 紐づく素材 |
|------|------|----------|
| 便益の正しい解釈 | 事実からどんな価値が導かれるか | |
| ポジショニング | N1の脳内マップのどこに一番乗りするか | |
| 唯一性 | 他の選択肢ではなぜこの便益が得られないか | |
| 自分ごと化 | なぜこの人に関係あると思えるか | |
| 証拠 | 事実として信じられる根拠は何か | |
| 便益の重なり | 複数の価値が同時に手に入るか | |

## 知覚コストを下げる

| 要素 | 問い | 紐づく素材 |
|------|------|----------|
| 金銭コスト | 何と比較して適正か（アンカリング） | |
| 金銭の正当性 | なぜこの価格で可能か（安かろう悪かろうの払拭） | |
| リスクコスト | 事実として何がリスク0か | |
| 比較コスト | 考えなくていい状態を作れるか（判断基準提示） | |
| 手間コスト | 行動の負荷はどれだけ小さいか | |
| 心理コスト | 最初の一歩のハードルは何か | |
| 時間コスト | どれくらいで結果が出るか | |

## やること
各要素の「紐づく素材」列を、訴求開発のアウトプットと素材リストから埋める。空欄は「この要素では戦えない」か「素材が足りない」のどちらか。
