# リサーチ

マーケティングはリサーチで8割決まる。

## マインドセット

ビジネス = 需要に対して供給を届けること。
**「誰に、どんな需要があって、どういう供給環境の中で、この商品はなぜ選ばれるのか？」**

人は商品が欲しいのではない。自分の現実世界に変化をもたらす「出力結果」が欲しいだけ。

## INPUT
- 案件名 / 商品情報 / LP等の情報ソース

## OUTPUT
- 商品の事実ツリー（源泉→派生。メリット+デメリット）
- 競合ジャンルの事実ツリー（同上）
- USP候補（競合が構造的に言えない事実）
- USP × Away/Toward の便益リスト
- N1定義（知覚トリガーまで含む）

---

## 3C分析フレームワーク

### Company（自社商品）— ②

**商品分析がコンセプトの質を決める。** コンセプトは発想力ではなく商品理解の解像度で決まる。

#### 商品理解の項目
商品名 / 価格 / 権威性 / 体験談 / 特徴 / 効果 / RTB / 使用感 / 機能的価値 / 情緒的価値 / オファー / 今買う理由 / ★解決する悩み（顧客のジョブ）

#### 死角チェックリスト（商品理解の完了前に必ず実行）
- □ 最も混同されやすい類似商品との技術的差分を調査したか？
- □ この商品ができないことを明確に言語化したか？
- □ 開発背景/創業ストーリーを調べ、ファクトチェックしたか？
- □ 最大のリスク/デメリット、ネガティブな口コミを調べたか？
- □ 既存ユーザーが「なぜ他ではなくこれを選んだか」を抽出したか？
- □ カテゴリーと隣接カテゴリーの境界は明確か？

#### 事実ツリー（源泉→派生）の作り方

商品の技術的特性（源泉）から派生する事実をツリー構造で整理する。
- 1つの源泉から複数の派生が生まれてOK
- 複数の源泉が組み合わさって1つの派生が生まれるケースも記載
- **メリットだけでなくデメリット（リスク・制約）も同じツリーに含める**
- 源泉と派生は全て「事実」であること。解釈や願望を混ぜない

**★因果チェーン検証（最重要）:**
全ての「→」について「なぜAがBを引き起こすのか」のメカニズムを説明できるか確認。
説明できないチェーンはWebSearchで検証する。「それっぽい」で書かない。

#### 競合ジャンルの事実ツリー

自社だけでなく、**関連する類似ジャンルの商品/治療のメリット・デメリットも同様のツリーで整理する**。
自社にないメリットを競合が持っている場合も正直に記載する。

#### USP候補の特定

事実ツリー（自社 + 競合）を見比べて、競合が構造的に言えない事実 = USP候補を洗い出す。
USPは事実。フレーミング（Away/Toward）を乗せない。

#### USP × Away / Toward の2レンズ

```
Away（不快の除去）: このUSPは誰のどんな不快を取り除くか？
Toward（快の獲得）: このUSPは誰にどんな快を新たに届けるか？
                   問題がない人でも得られる快は何か？
```

同じUSPでもAwayとTowardで全く違うN1とコンセプトが出る。

#### 便益の4軸整理

- 軸1 性質: 得るもの / 失わないもの / 続くもの
- 軸2 深さ: Lv.1機能 → Lv.2生活 → Lv.3自己 → Lv.4社会
- 軸3 時間: 検討時 / 体験中 / 直後 / 長期
- 軸4 比較: vs各競合 / vs代替手段 / vs何もしない

### Customer（市場・顧客）— ③④⑤

#### ③ ターゲット選定
USPの便益（Away / Toward）を持っている人はどんな層か。
→ TOTで複数仮説を並列生成し、5軸で評価:
能力適合 / 便益共鳴 / 市場規模 / 到達可能 / 競合空白
**能力適合がNGなら即却下。**

#### ④ N1選定（最重要）
ターゲット範囲の中から、たった1人の具体的な人間を選ぶ。
**★ ほとんどの広告代理店はここが浅い。ターゲット選定だけで止まる。**

#### ⑤ N1の情報整理
- どんな生活を送っているか
- どんな悩みを持っているか / どんな理想を持っているか
- その背景は何か
- 悩みを解決する方法として何を認知しているか（各解決策のメリット・デメリットの認知も含む）
- 何に対してどれくらいのお金を払い、その背景は何か
- **知覚トリガー**: どんな場面で快不快反応が発火するか？

### Competitor（競合）— ⑥
→ 詳細は `競合分析.md` で実行
- **N1の脳内にある競合する解決方法の中で、必ずNo.1になれるポジションを設定する**
- たとえ似た商品があっても、N1にとってのOnly1は絶対ある

## マーケット知識
- そのマーケットの専門知識が極めて重要
- マーケットの境界を再定義する視点が新しいコンセプトを生む

## 3つの目
- 鳥の目（市場全体）
- 虫の目（N1の深掘り）
- 魚の目（流れ＝変化＝新しい需要）
- すべて需要の観点で捉える

## 思考の型
- **COT**: 順序立てた分解。商品事実の列挙で使う
- **TOT**: 複数仮説を並列生成して比較評価。ターゲット選定・N1選定で使う
- **GOT**: 複数視点の統合。コンセプト生成で使う
- **Clarifying Questions**: 死角の発見。商品理解の完了チェックで使う
