# 訴求開発

## INPUT
- リサーチデータ（②〜⑥の全成果物）

## OUTPUT
- コンセプト（便益×カテゴリー）
- N1定義（知覚トリガーまで含む）
- WHO × WHAT × WHY

---

## コンセプト

- コンセプト = **便益 × カテゴリー**の掛け算で生まれる新しい便益の表現
- カテゴリーの再定義が鍵（水→飲む化粧水、債務整理→利息の乗り換え、矯正→骨格美容）
- 良いコンセプト: 一言で「欲しい」「知りたい」と思える。イメージが広がる

### コンセプト開発の行き来
線形ではない。便益→N1→コンセプト表現を行き来する:
1. 商品の便益から「こんな人が欲しがりそう」と仮説を立てる
2. その人の解像度を上げる（N1定義）
3. N1の理解が深まるとコンセプトの言葉が変わる
4. 人の発見から入ることもある

### コンセプトワード
- 一語でなくてよい。便益を重ねて「一言で欲しくなる」を達成する
- 便益×便益×カテゴリーで贅沢感（例: ダウンタイム0の骨格矯正）
- NG: 便益不明 / カテゴリーが嘘 / 新しさなし / 説明的すぎ
- **NGワード**: N1がマイナスイメージを持つ言葉は使わない（脳内マップから導出）

### コンセプト生成の方法（GOT: 4インプットの交差点）

コンセプトは以下の4つを**全て同時に満たす交差点**にだけ存在する:

```
INPUT A: 商品の原点（なぜ生まれたか。開発者の哲学）
INPUT B: N1の欲求構造（何を得たい × 何を失いたくない × 知覚トリガー）
INPUT C: 競合の空き（誰も言っていない × N1が求めている）
INPUT D: 便益の本質（最上位の組み合わせ便益）
```

部分的にしか満たさないコンセプトはNG。4つの交差点を探す。

**成功例の構造:**
- 飲む化粧水 = 馴染みの便益(化粧水) × 意外な手段(飲む) → 「え？」が生まれる
- 骨格美容 = 結果(骨格レベルの変化) × 馴染みカテゴリ(美容) → 大きさが伝わる
- 利息の乗り換え = 馴染みの行為(乗り換え) × 意外な対象(利息) → 「できるの？」が生まれる

## ポジショニング

- N1の脳内マップ上のどこに自社商品を置くか
- **一番乗り以外に価値はない。2番目は忘れ去られる**
- 脳内マップは自社都合で作るものではない。ユーザーの脳内に既にある
- 手順: ①脳内マップを知る → ②空きを見つける → ③一番乗りする
- N1の「解決策 - 知っていることと印象」が脳内マップそのもの
- **必ずNo.1になれるポジション（Only1）を設定する。似た商品があっても、N1にとってのOnly1は絶対ある**

### 知覚トリガーについて
購買行動は 知覚 → 快不快反応 → 感情 → 行動 のパターンで起きる。
N1の原体験に紐づく「知覚トリガー」を特定することで、CRで与えるべき知覚が明確になる。
例: 「写真に写った自分の歯を見た瞬間」→ 不快（恥ずかしい、隠したい）→ 行動（口を閉じて笑う）

## WHO × WHAT × WHY

- **WHO 3軸**: 状態 / 認知状況 / 制約条件
- **WHAT 5つの価値軸**: 機能 / 情緒 / 金銭 / 信頼 / 体制 → 提供便益 → 便益提供シーン
- **WHY**: 便益のない現状シーン（知覚トリガーが発火する場面）→ 強いペイン/願望 → だから刺さる
- WHOごとにWHATは変わる。1コンセプトに複数WHO

## 欲求ファネル

| ファネル | 定義 |
|---------|------|
| 潜在 | 課題をまだ捉えていない |
| 準顕在 | なんとなく感じているが向き合っていない |
| 顕在 | 課題として認識し向き合っている |
| 比較検討 | 具体的な商品を検討/使用中 |
