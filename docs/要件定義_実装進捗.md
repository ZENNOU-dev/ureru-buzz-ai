## できていないところ（要件定義上のギャップ）

以下は **GitHub のイシュー** として起票する想定（ラベル: `要件ギャップ`）。詳細は各イシュー本文参照。

1. **オーケストレーター・承認・フェーズ遷移** — `orchestrator` はプレースホルダ。ワークフローエンジン、Notion 承認ポーリング、Slack 通知、差し戻しは未実装。
2. **Phase 1 案件セットアップ** — AP/MP 入力項目の完全 UI・バリデーション・Notion 永続化・`SetupAgent` なし。
3. **Phase 2〜5 制作エージェント** — `ResearchAgent` / `AppealAgent` / `PlanningAgent` / `ScriptAgent` および Web 検索・広告ライブラリ等のスキル連携なし（画面のみまたは未接続）。
4. **Phase 6 編集概要の完成度** — 動画名シート準拠の全項目（音声注釈、レギュ欄、SE/ME、エフェクト、修正ループ等）・Slack 連携・本番品質の Notion マッピングは途中。
5. **素材マッチ・3 軸・本番パイプライン** — 要件の「過去実績 / 市場データ（動画広告分析 Pro）/ コンテンツマッチ」完全統合、Vertex・フレーム抽出の運用化は未完了。
6. **Phase 7 Remotion・動画生成** — `remotion-project` パッケージなし、`VideoAgent` / `RemotionSkill` / TTS 本番経路なし。
7. **Phase 8 入稿** — Meta/TikTok/YouTube 等への API 入稿、`SubmissionAgent`、動画名入力→自動入稿フローなし。
8. **Phase 9 運用・データ取得** — 5 分間隔ジョブ（BullMQ/Redis）、媒体パフォーマンス取り込み、`OperationAgent` の判断実行なし（DB テーブルは存在）。
9. **Phase 10 レポート・分析** — `ReportAgent` / `AnalysisAgent`、日次/週次/月次レポート自動化、BigQuery 連携、マクロ/ミクロ分析の実装なし（レポート UI はモック寄り）。
10. **レギュ・チャット横断** — `RegulationAgent` / 自動注釈、レギュナレッジ検索の本番組み込み、チャットと配信データ・ナレッジの API 統合なし。
11. **共通スキル・非機能** — `DriveSkill` / `SlackSkill` / `ApprovalSkill` / 各種広告 API スキル、Cloud Run・Scheduler・CI/CD、コミュニケーション自動取り込み（技術要件書 9 章相当）は未整備。
12. **Tier 5 管理系** — ガント・請求・目標管理・制作ステータスと Notion/DB の本番連動は未着手（要件 6〜8 章）。

## イシュー一括作成

PAT に `workflow` スコープがない環境では `.github/workflows` のプッシュが拒否されるため、**ローカル（または別 CI）から GitHub API で作成**する。

```bash
cd /path/to/ureru-buzz-ai
export GITHUB_TOKEN='...'   # Fine-grained: Issues Read and write
node scripts/create-requirement-gap-issues.mjs
```

既に同じタイトルで `要件ギャップ` ラベル付き Issue がある場合はスキップする。
