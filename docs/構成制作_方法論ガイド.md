# 構成制作

## INPUT
- コンテンツ開発のアウトプット（表現手法/フック型/バランス設計）
- セールス設計図（知覚価値×知覚コスト、素材紐づけ済み）

## OUTPUT
- パート順序と各パートの目的・使用素材

---

## セールスの核の配置

- **セールスの核 = セールスのフック（入口）。** 構成の真ん中ではなく冒頭寄り
- 核でグッと引きつけ → 知覚価値最大化 → 知覚コスト最小化 → CTA

### 核の前に何を置くか（N1の認知状態で変わる）
- コンセプト自体が強い → 核から始まる
- 核だけだと信じてもらえない → 権威性を先に出す（「NHKで放送された〜」）
- N1が潜在層 → 自分ごと化を先に出す（悩み描写→核）

毎回同じ順番ではない。

## 商品名の登場タイミング

- **商品名を出す前に教育パートを置け。** 売り手の文脈で教育を受けると信じてもらえない
- 教育（コンテンツとして受け取る）→「じゃあどこで？」→ 商品登場

## 認知転換の演出

- **講義ではなくREVEAL。** 「知ってました？」は広告テンプレ感で離脱
- **驚きが先、理由が後。** 結果を見せる→「え、何これ？」→種明かし
- 視聴者が「知りたい」状態になってから教育する → 教育がコンテンツになる

## 構成テクニック

- **緊張→解決で唯一性を入れる。** 「ただし〜が必要」で不安にさせてから解決。価値が上がる
- **アンカリング。** 高い選択肢を先に出してから安い選択肢
- **オファー積み。** 複数オファーを「これでは終わりません」で重ねる
- **CTAのハードル最小化。** 「チェックだけでも」「後から変更OK」

## 各パートの目的（4つの壁との対応）

| パート | 壁 | 目的 |
|--------|----|----|
| フック | 1a | スクロール停止 |
| つなぎ | 1b | ボディへの橋渡し。教育自体が見続ける理由にもなる |
| 教育+信頼構築 | 2 | 信じさせる。認知転換+証拠 |
| オファー+CTA | 3 | 今やる理由を作る |
